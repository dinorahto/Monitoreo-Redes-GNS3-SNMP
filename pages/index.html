  • Protocolos, API's, Documentación y Software

    Resumen

    Se hizo uso de diferentes softwares, API’s y protocolos, los protocolos más usados fueron: DHCP, SNMP y UDP. API’s: Net-snmp, simple-snmp de Node y Boostrap. Software: TunTap, Gns3, Sublimetext y VirtualBox.

  • Virtualización

    Se decidio que debían ser pocos host por que cada Host debe ser una maquina virtual las cuales debían estar corriendo en el momento de las pruebas, cada una cuenta con un IP diferente.

    TunTap, y la virtualización para crear interfaces simples, es un pequeño pedazo de software que permite tener varios adaptadores lógicos. Puede asignar direcciones IP a estos adaptadores y generalmente los utilizan para pruebas.

  • Routers & Switchs

    Los Routers de GNS3 te permiten mantener más de las dos imples interfaces que estan por default, para lograr esto es preferible dar de alta las cuentas de VLAN.

    La consola de los Routers acepta terminos abreviados.

    Comandos más comunes: config t, ip address X.X.X.X X.X.X.X, interface fa?/?

  • SNMP

    Simple Network Management Protocol es un protocolo de la capa de aplicación que facilita el intercambio de información de administración entre dispositivos de red. Los dispositivos que normalmente soportan SNMP incluyen routers, switches, servidores, estaciones de trabajo, impresoras, bastidores de módem y muchos más. Permite a los administradores supervisar el funcionamiento de la red, buscar y resolver sus problemas, y planear su crecimiento

  • Implementación

    Existen muchos API's que te dejan jugar con los procesos que genera SNMP muchos de ellos estan escritos para poder ser usados en Javascript, Node.js y PHP

    Los calculos son largos y requieren de crear diferentes funciones dependiendo de las necesidades

  • Interface grafica

    Al igual que hay API's de software, tambien hay API's de diseño que te permiten hacer diseños mucho más rapidos y sencillos en corto tiempo.

    Aún asi el diseño es muy laborioso

  • Conclusiones

    Los calculos y graficas de este sistema, se volvieron laboriosos y complicados por los diferentes Hosts y diferentes comandos que se podian usar en SNMP

    La cantidad de archivos generados es demasiada.

    Existen muchos sistemas en el mercado tanto per cost como Open Source que realizan todo el trabajo sin tener que ingresar datos de IP
